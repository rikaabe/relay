# Relay — 案件管理アプリ UIモックアップ

バイク・車の査定/買取業務における案件管理を、Slack + admin の運用から置き換えるための自社アプリ「Relay」のUI/UXモックアップです。

## 📱 モックアップを見る方法

**いちばん簡単な方法**：`caseflow-mockup.html` をダウンロードして、ダブルクリックでブラウザで開くだけです（インストール不要・単一ファイルで動きます）。

ローカルサーバーで見る場合（開発者向け）：

```bash
node serve-mockup.mjs
# → http://localhost:7891 で開く
```

## 🎯 このアプリで実現すること

現在のSlackチャンネル運用の課題（案件割り当てが早いもの順で不公平・通知の見落とし・対応状況がadminを開かないと分からない）を解決します。

- CVの自動振り分け（受信が古い案件から順に処理・担当者は負荷や担当区分で決定）
- 振り分け時に担当者へ通知（通知内にadminを別タブで開くボタン付き）
- 案件ごとのスレッドで引き継ぎ・相談（@メンション対応）
- 対応スピード・対応件数が分かるKPI画面
- 自動振り分けロジックを操作できる設定画面
- バイクと車を分けて管理しつつ、全員が両方の状況を把握できる一覧

## 🖥 画面構成

全員が同じ管理コンソールを使います（モード分けなし）。操作はすべて**左のサイドバー**に集約し、視線と手の移動を最小化しています（上部バーは検索のみ）。サイドバー最下部には自分の名前・アイコンがあり、クリックでマイページ（スタンプ変更・稼働状況・パフォーマンス）が開きます。

| 画面（左タブ） | 内容 |
|------|------|
| **マイ案件** | 開いたとき最初に表示。上部の進捗バーで「今日の担当 ◯件中 ◯件対応済み」が一目で分かる。下に自分の未対応案件の一覧 |
| **通知** | 通知はすべてここに一本化（右上のベルは廃止）。その場でスレッド返信・admin遷移・通知設定へのショートカット |
| **バイク/車 案件一覧** | 受信が古い順の全案件。対応済みもグレーで残る |
| **KPI** | 対応状況の可視化 |
| **設定** | 自動振り分け・チーム編成・通知設定 |

## 🔑 設計のポイント

- **「対応済み」はRelay側で操作しない**：admin側の操作（お客様へのメッセージ送信・承認して入札開始）に自動連動する設計。Relayのための追加作業はゼロ（「▶ デモ」ボタンでこの動きを再現できます）
- **案件番号はadminのURL末尾番号**（例：555555）に統一。現場の呼び名とそのまま一致
- **対応済み案件は一覧にグレーで残る**：誰が見ても状況が分かる（現状はadminを開かないと分からない）
- **経過2時間超の案件は行/カードが黄色**になり対応漏れを防止
- ホバー操作なし・ボタンは常時表示・文字ラベル併記（PC操作に不慣れなメンバーへの配慮）
