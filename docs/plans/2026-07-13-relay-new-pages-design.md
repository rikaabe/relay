# Relay 新ページ6機能 設計（2026-07-13）

tel案件中心だったRelayに、以下6機能を追加する。ナビは「グループ見出し付きサイドバー（案A：フル分離型）」。

## サイドバー構成

```
◾ 案件    : マイ案件 / 通知 / バイク案件(tel) / 車案件(tel) / コメント案件 / 交渉依頼
◾ ツール  : リマインダー / 掲示板 / IDストップ
◾ 分析    : KPI / 目標
フッター  : マイページ／設定（現状のまま）
```

## 1. コメント案件（バイク/車タブ）

- 現運用：Slack #bike_101_upload_comment 等に「ユーザーからメッセージが届きました／担当者／内容確認ページURL／メッセージ内容」が流れる。これをページ化。
- 担当者は既にtelで決まっている → その担当者に通知し、マイ案件にも入る。
- 担当者不在時（ハイブリッド方式）:
  1. カードに「代打募集」バッジ。チームの誰でも「対応します」で引き取れる。
  2. 一定時間（初期30分）拾われなければ出勤中メンバーへ自動振り分け（「代打」バッジ）。
- 対応済みは既存方式（チェックアイコン＋グレーアウト）。

## 2. 交渉依頼

- 現運用：Slack交渉チャンネル投稿 → katix-cs-board.web.app 自動取込 → 交渉担当対応 → 完了 → 依頼者にメンション。これを1ページに置換。
- 「＋交渉を依頼」：店舗名/案件URL/依頼内容を登録。
- ステータス：未着手 → 対応中 → 完了。完了時に依頼者へ通知。バイク/車チップ。

## 3. リマインダー

- admin側で設定されたリマインダーの受信一覧（Relayは受け側のみ）。
- 項目：予定時刻・対象案件URL・内容・設定者・宛先。
- セクション：本日／今後／完了済み。時刻到来で通知タブ＋バッジ。「確認した」で完了へ。

## 4. 掲示板

- タブ：お知らせ（リーダー/管理者投稿・ピン留めで上部固定）／自由投稿（全員）。
- 投稿＝タイトル＋本文＋カテゴリ。コメント・簡易リアクション・未読バッジ。

## 5. IDストップリスト

- スプレッドシート「現金書留・IDストップ」の置き換え（ウェルネットID発行ストップ案件）。
- 列：入力日／案件URL／担当／備考（理由）／状態。
- 「発行していいよ」チェック＝解除（グレーアウトで下へ）。フィルタ（ストップ中のみ/すべて）＋検索＋追加。

## 6. 目標

- 月次目標＝成約数・成約率（個人ごと）。
- ①今月の目標と実績（成約数・成約率をリアルタイム表示）
- ②telシミュレーター：直近成約率×残り営業日から「1日◯件telで月間成約◯件」を逆算。スライダー連動。
- ③日次tel目標カレンダー：月1回「1日◯件」を設定し、日ごとの達成/未達を色表示。
- マイページに現在の成約率をリアルタイム表示。

## 共通

- 通知は既存の通知タブに種別アイコン付きで統合（コメント代打・交渉完了・リマインダー・掲示板お知らせ）。
- マイ案件は tel＋コメント＋交渉 を統合表示（種別チップ＋代打バッジで区別）。
- データは全てモック（JS定数）。既存tel案件・KPI・内部レートには手を入れない（サイドバー再編のみ）。
- 実装先：caseflow-mockup.html（1ファイル）。
